1. Introduction

1.1 What is Dev Toolkit?

Dev Toolkit is a modular, front-end-focused development toolkit designed to streamline and standardize the setup, scaffolding, and coding experience across multiple projects. It includes a command-line interface (CLI), configurable project templates, reusable UI components, code formatting tools and optional integrations with frameworks and libraries like React, Vue, Tailwind CSS, TypeScript and more.

1.2 Why Use Dev Toolkit?

  • Faster Setup: Instantly scaffold projects with pre-configured environments.
  • Reusable Assets: Use pre-built components, hooks, and utility functions.
  • Customizable: Easily adapt to your preferred tech stack or team standards.
  • Productivity-Boosting CLI: Automate repetitive tasks like component generation, linting, formatting, and more.
  • Consistent Structure: Enforces a clean and consistent folder layout and naming conventions.

1.3 Who is it for?

  • Freelancers who want to quickly spin up consistent, production-ready projects.
  • Agencies/Teams looking for a standardized internal tool to reduce onboarding time.
  • Educators or course creators needing consistent starter templates for front-end assignments.
  • Open-source developers maintaining multiple projects with shared setup needs.

1.6 Example CLI Usage

npx dev-toolkit init my-app --template react-tailwind

This command initializes a new project named my-app with a React + Tailwind CSS setup, including routing, pre-configured ESLint and Prettier rules, as well as a default folder structure.

2. Quick Start

2.1 Overview

This section will walk you through installing Dev Toolkit, initializing your first project, and using the CLI to scaffold your first components. By the end, you will have a fully functioning starter project with best practices already in place.

2.2 Prerequisites

To use Dev Toolkit, ensure you have the following installed:

  • Node.js (v16 or higher recommended)
  • npm or yarn
  • Terminal or Command Prompt access
node -v npm -v

üß†Tip: Use nvm or fnm to manage Node versions.

2.3 Installing Dev Toolkit

You can use Dev Toolkit via npx (no global install needed), or isntall it globally if you will be using it frequently.

  • Option 1: Use with npx (no install)

    npx dev-toolkit init my-project

  • Option 2: Install globally

    npm install -g dev-toolkit

  • To verify installation:

    dev-toolkit --version
Available options
Flag Description
--template Choose a starting stack (e.g., react-ts, vue-basic, svelte-min)
--features Comma-separated list: tailwind, eslint, prettier, husky, etc.
--git Initialize a Git repo automatically
--install Auto-install dependencies after scaffolding

‚úÖ If no template is provided, you'll be prompted interactively.

2.6 Using the CLI to Scaffold Files

Once inside your project, use the Dev Toolkit CLI to quickly generate components, hooks, utilities, pages and services to accelerate your development.

For example, you can create a new button component or a custom hook with a single command.

Example: Generate a new UI component

dev-toolkit generate component Button

Creates the following files:


                                    src/components/Button/

Example: Generate a custom hook

2.7 Run the Project

Once scaffolded, run the project using your framework's dev server.
If you are using React (default for most templates):

npm run dev

Dev Toolkit auto-generates dev, build, format scripts in package.json.

2.8 Optional: Format & Lint the Code

# Format all files using Prettier dev-toolkit Format
# Run ESLint dev-toolkit lint

These commands use pre-configured rules included with your template unless overridden in .devtoolkitrc.
