1. Introduction

1.1 What is Dev Toolkit?

Dev Toolkit is a modular, front-end-focused development toolkit designed to streamline and standardize the setup, scaffolding, and coding experience across multiple projects. It includes a command-line interface (CLI), configurable project templates, reusable UI components, code formatting tools and optional integrations with frameworks and libraries like React, Vue, Tailwind CSS, TypeScript and more.

1.2 Why Use Dev Toolkit?

  • Faster Setup: Instantly scaffold projects with pre-configured environments.
  • Reusable Assets: Use pre-built components, hooks, and utility functions.
  • Customizable: Easily adapt to your preferred tech stack or team standards.
  • Productivity-Boosting CLI: Automate repetitive tasks like component generation, linting, formatting, and more.
  • Consistent Structure: Enforces a clean and consistent folder layout and naming conventions.

1.3 Who is it for?

  • Freelancers who want to quickly spin up consistent, production-ready projects.
  • Agencies/Teams looking for a standardized internal tool to reduce onboarding time.
  • Educators or course creators needing consistent starter templates for front-end assignments.
  • Open-source developers maintaining multiple projects with shared setup needs.

5. Component Library

Introduction to Component Library

The Component Library in Dev Toolkit is a collection of reusable, accessible and theme-ready UI components designed to help you build interfaces faster and more consistently. These components can be used as-is or customized to match your project's identity.

Whether you are building a dashboard, marketing site, or application UI, the built-in components save time and reduce repetition.

5.1 Overview of Component System

The library is modular and lightweight. Components are:

  • Built with accessibility best practices (ARIA, keyboard navigation)
  • Theme-aware (supports CSS variables or Tailwind)
  • Compatible with TypeScript or JavaScript
  • Styled using CSS Modules, SCSS, or Tailwind depending on your config
  • Automatically documented via IntelliSense (if TypeScript is enabled)
  • Tree-shakeable to keep your bundles lean
  • Easily customizable via component props and theming

All components are lacated in the dev-toolkit/ui package after initialization and can be imported individually.

For example:


                                                    
                                                    import { Button } from 'dev-toolkit/ui';

                                                    function MyComponent() {
                                                        return <Button variant="primary">Click me</Button>;
                                                    }

6. Utilities

Utilities Overview

The Dev Toolkit Utility Library includes a set of ready-to-use, framework-agnostic helper functions that cover frequent development needs. These utilities are lightweight, tested and follow predictable naming conventions to encourage consistency across projects.

Designed to work with React, Vue, Svelte or vanilla JavaScript projects. Utilities can be imported directly from dev-toolkit/utils or generated using the CLI.

Whether you are formatting data, manipulating arrays, or working with the DOM. These helper functions allow you to write cleaner, faster code with minimal setup.

6.1 How to Use Utilities

Location of Utilities

Utilities are located in the src/utils/ directory by default and are written in either JavaScript or TypeScript based on your project setup.

Each utility is modular, tree-shakeable and designed for easy reuse across your app's logic, UI behaviors and API interactions.

Example: Using a Utility Function


                                                    import { debounce } from 'dev-toolkit/utils';

                                                    const handleInput = debounce((vlaue) => {
                                                        console.log('Searching for:', value);
                                                    }, 300);

This debounce function waits until the user stops typing for 300ms before executing the search logic - great for performance in input-heavy UIs.

Generating Your Own Utility

dev-toolkit generate util slugify

7. Hooks

Hooks Overview

The Dev Toolkit Hooks provides reusable logic abstractions for managing state, effects and more within your components. Designed to be lightweight and flexiable, they help you streamline your React workflows with minimal boilerplates.

Whether you need to manage async data fetching, handle form state or add custom behaviors, Dev Toolkit hooks are built with performance and ease of use in mind.

Dev Toolkit provides a set of custom React hooks that simplify state management, storage, user input and more. These hooks are designed to follow best practices for reusability, composability and accessibility in modern React applications.

Hooks can be scaffolded using the CLI and are automatically placed in the src/hooks directory by default. Each hook is fully typed if TypeScript is enabled and includes optional test scaffolding if required.

7.1 What are Custom Hooks?

Custom hooks are JavaScript functions that use built-in React hooks (like useState, useEffect, etc.) to encapsulate and reuse logic across components.

They allow you to bundle related logic that an be shared across multiple components, making your codebase DRY (Don't Repeat Yourself) and easier to maintain and test.

Why use them?

  • Extract complex logic from components
  • Keep components focused on rendering
  • Promote code reuse and separation of concerns
  • Improve readability and maintainability

For example, a useFetch hook can encapsulate API calls and loading states, allowing components to consume data without managing the underlying details.
